# Launch Channel Guide

## Tier 1: Always-on for significant launches
- **Own Slack channels** — #announcements, product channel. First and loudest. Schedule ahead.
- **Email to existing customers/list** — direct, measurable, high-trust. Required for anything that changes the product.
- **Product changelog / release notes** — permanent record; cross-link from other channels.

## Tier 2: For external launches
- **Blog post** — SEO value, linkable asset, shareable. Write the post before you write the tweets.
- **Social (LinkedIn, Twitter/X)** — amplify the blog. Not a substitute for it.
- **Product Hunt** — for developer/startup audiences; requires coordination (top hunters, timing, demo GIF). High effort, optional.

## Tier 3: For major launches only
- **Press / analyst outreach** — requires embargo management, press release, PR contact. Only worth it if you have a real news hook.
- **Webinar / live demo** — high effort but high-quality pipeline for enterprise features.
- **Paid amplification** — boost only posts with organic signal; don't pay to promote a cold post.

## Sequencing principle
Seed internal and partners first (T−2 to T−1 days) → coordinated external blast on launch day → sustain with follow-up content (T+3 to T+7).

## Small feature launches (don't over-engineer)
For minor improvements: changelog entry + one Slack post to users. Full brief optional.
