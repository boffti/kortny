# Pricing Models Reference

## Value metric options (pick the one that scales with customer value)

| Value metric | Best for | Caution |
|---|---|---|
| Per seat / per user | Collaboration tools, team software | Customers reduce seats aggressively; fight per-user pricing if usage varies |
| Per usage unit | API, infrastructure, AI inference | Revenue fluctuates; hard to forecast; can create anxiety about cost |
| Per outcome | Results-based (e.g. per lead generated) | Powerful alignment; hard to measure; complex contracts |
| Flat / site license | Enterprise, security-sensitive buyers | Easy to sell; leaves money on table for high-usage customers |
| Tiered by features | Most SaaS | The default — works when feature differentiation is clear |
| Tiered by usage limits | Freemium models | Works when the upgrade trigger is natural and the limit is felt |

## Tier structure principles

**Three tiers is usually right.** Two is too little anchoring; four or more creates decision paralysis.

- **Free / Starter**: Demonstrate core value. No premium features. Upgrade trigger must be encountered naturally.
- **Core / Growth**: Where most SMB customers land. Price to be a no-brainer for the ICP.
- **Pro / Enterprise**: Removes limits, adds admin/compliance/SSO features. Price to support a sales conversation.

## Pricing psychology
- **Left-digit effect**: $99 feels much cheaper than $100.
- **Anchor high**: Show the most expensive tier first — it sets the reference frame.
- **Annual discount**: Offer 20-30% off annual to improve cash flow and reduce churn; don't go below 15%.
- **Add-on vs. bundle**: If a feature appeals to <50% of users, offer as an add-on. If >50%, bundle it.

## Common mistakes
- Pricing too low for B2B (signals low quality, attracts wrong customers).
- Pricing based on cost-plus instead of value.
- Too many tiers — causes analysis paralysis.
- No upgrade trigger in the free/starter tier.
- Feature differentiation that customers don't care about.
