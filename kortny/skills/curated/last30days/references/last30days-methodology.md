# Last 30 Days Briefing Methodology

## Briefing structure

### [Subject] — Last 30 Days
*Period: [Start date] – [End date]*
*Researched: [Today's date]*

### Timeline of key events
List events chronologically. Each event: date + what happened (one line). Flag events you could not verify with the date with "date unconfirmed."

### What changed
Bullet the most significant shifts — product, strategy, leadership, market position, public perception.

### What stayed the same
Brief note on what's notably unchanged (useful when the question implies expected change that didn't happen).

### What's unclear or contested
Items where sources conflict or where the full picture isn't available yet.

### So what (the verdict)
One sentence: what does this period's events tell us about the trajectory of [subject]?

---

## Search strategy (run these if web search is available)

Run 3-5 targeted searches for better coverage than one broad search:

1. `"[subject]" news [month year]`
2. `"[subject]" announcement OR release OR launch [month year]`
3. `"[subject]" [specific angle — funding / controversy / product / leadership]`

Date-filter results to the stated period. Discard anything outside it even if relevant — note it as context instead.

## Quality bar for inclusion

Include: confirmed events with a date, from a named publication or official source.
Exclude: rumors without sourcing, undated blog posts, events outside the date window, opinion pieces without factual content to extract.

## When web search isn't available

Work with what's in the thread context and Kortny's knowledge graph. State clearly: "Based on available context only — web search not used. Verify key claims independently."
